Χ</font><br><strong>3. 准备</strong><br><strong>3.1 改简历，拿面试</strong><br>海投了30个左右，有两个面试，其他都被拒了或者没后续。其他面试都是recruiter reach out的。楼主i人，没有找过内推。Recruiter reach out最有用，我开了LinkedIn会员 + open to work(only to recruiters), 加了一些skills tag，大概2周后慢慢有recruiter reach out，前一个半月是高峰期，后面就没有了。<br><strong>3.2 刷题</strong><br><strong>TL;DR: </strong>刷题刷的是类型，看到一道题能识别类型，知道用什么数据结构和方法去解决，把常见类型刷通了就差不多能应付大部分coding轮了。刷胖头龙分类list理解常见类型 + 刷Meta last 30 days list + 用Jiakaobo + chatgpt辅助理解 + 总结面试框架 + 看地里面经。<br>因为没有系统刷过题(8年前找实习的时候刷过200多题)，一开始有很长的cold start，去年12月到今年2月，每天下班后刷个两三题就累了，刷了也感觉脑子空空，需要边吃话梅边刷对冲痛苦。<br>4月有朋友拿了好几个offer，我想他的方法应该是proven work的，就开始刷他推荐的胖头龙分类list，大概两个月后刷完大部分，刷完掌握了常见类型，脑子里大概有个谱了，什么时候用2 pointer，什么时候priority queue，什么时候stack，什么时候backtracking, 什么时候dfs, bfs, graph, dp, etc。6月中开始刷Meta的last 30 days tag题，反复刷了很多遍，每题都总结一下pattern，搞明白time/space complexity，不obvious的题会写写thought process。dd的tag题很少但都很经典，推荐做做。7月中到7月底店面，大部分能过。<br>不会的题，我就找刷题博主Jiakaobo的讲解 + ask chatgpt，prompt - paste code + explain this to me/walk me thought process for this, 反正哪儿不会问哪儿，特别好用。<br>面试前要总结一个自己的coding轮框架和language，我的大概是这样：<ul><li>Read the description and clarify question (比如are the value unique? is the array sorted? etc)</li><li>Discuss solution (没有思路的时候，可以把problem break down steps，然后discuss该用什么数据结构, time/space complexity大概是怎么样)</li><li>Write code</li><li>Check if miss boundary check and add logs for debugging</li><li>Run and debug</li><li>Discuss time/space complexity and optimize if needed</li><li>Fix coding style if still have time</li></ul><br>以上提到的资料：<ul><li>胖头龙分类list - <a href="https://link.1point3acres.com/?url=https%3A%2F%2Fgithub.com%2Fxxiaomm%2FLeetCode%2Fblob%2Fmain%2F%25E8%2583%2596%25E5%25A4%25B4%25E9%25B1%25BC%25E7%259A%2584list.md">🔗 github.com</a></li><li>Jiakaobo - <a href="https://link.1point3acres.com/?url=https%3A%2F%2Fwww.jiakaobo.com%2Fleetcode.html">https://www.jiakaobo.com/leetcode.html</a></li></ul><br><strong>3.3 SD</strong><br>(以下staff+ 的朋友们可略过)<br>SD刷的也是类型和理解，得真的理解和懂才能讲得明白。我主要看了hello interview，第一遍看了视频还是云里雾里，第二三遍看了文字版，结合问chatgpt去理解，这个系统的non-func requirement是啥，怎么实现它？这是什么类型的问题？(常见的类型比如streaming processing, double <a href="https://link.1point3acres.com/?url=http%3A%2F%2Fwww.awin1.com%2Fawclick.php%3Fmid%3D6776%26id%3D423189">booking</a> ，real time update, leaderboard, pub/sub, post feed, media upload/download, media streaming, payment etc)，这其中的tradeoff是什么？怎么scale？另外就是常用产品的core features和scaling capability也要弄明白，比如Redis, SQS, Kinesis, Flink etc。<br>SD是店面结束后8月初开始准备的，大概准备了1个月，看了两三遍hello interview + 一些面经题，常考的题在excalidraw上自己画了一遍，有些面经题自己做了我会让chatgpt点评一下，非常受用。除了hello interview也结合看了其他网站(google和youtube搜题目就行)。<br>最后就是形成自己面试时的做题框架，拿到一道题，functional requirement是面试官给的，non-func requirement大概就几个角度(low latency, high scalable, availability, consistency, Idempotency, accuracy)，你决定prioritize哪几个，可以和面试官确认一下。 Requirement理清楚了，就discuss data storage和core component，然后开始画high-level design，识别类型，用对应的解决方案去satisfy requirements。<br>以上提到的资料：<ul><li><a href="https://link.1point3acres.com/?url=https%3A%2F%2Fwww.hellointerview.com%2F">https://www.hellointerview.com/</a></li><li>另一个类似helloInterview网站, topK讲的不错 - <a href="https://link.1point3acres.com/?url=https%3A%2F%2Fblog.algomaster.io%2Ft%2Fsystem-design">https://blog.algomaster.io/t/system-design</a></li><li>Youtube东欧大哥，rate limiter讲的不错 - <a BQ</strong><br>BQ能分享的不多，时间有限，是onsite前两天准备的，一天用STAR写了一下常见问题的story(conflict; cross-team collaboration etc)；一天总结了一下project dive deep，project impact, project complexity(architecture/challenges), people complexity。<br><strong>3.5 时间安排</strong><br>卷组划不了水，中间接了一个cross-org的项目，除了technical stuff还有很多alignment和meeting要drive，9:30上班，6:30下班，能不加班就不加，不加班的时候吃完饭玩会手机，九点到凌晨两点准备，看累了就刷会手机吃点东西，洗个澡能精神点。周末睡到2点，下午和晚上准备。这样大概准备了五个月，社交和健身也停了，每周靠火锅，massage和睡懒觉续航。<br>在卷组，工作和跳槽没法兼顾，只能保证正常输出，perf是会下滑的，做不了high-end perf，所以一定要想清楚为什么要跳槽，是occasionally looking还是一定要走，跳槽是一个耗时耗力的investment。一开始工作60%，跳槽40%，后面面试多了准备跳槽的比例就可以多一点。<br>全程在职准备，没有请假；有两个店面请假了一天，其他是中间溜出去；onsite请了两周假集中面。<br><strong>3.6 其他Tips</strong><ul><li>面试前建议多看地里的面经，dd onsite的题全是面经里的，meta也很多面经题。</li><li>面试时要自信，气场也是一部分，一个小方法是提高音量，降低语速。</li><li>非BQ轮自我介绍尽量简短，特别是Meta的面试只有45min，自我介绍讲太多就没时间做题了。</li><li>在职的话建议在公司面，比较professional，最好是那种先到先得的room，meeting room前后有人schedule的话用起来很匆忙，room尽量选离过道/电梯远的地方。</li><li>我一般一天最多排三场，多于三场分两天完成。每场之间建议隔个15min或者30min，一来以防没答完微超时，二来可以去窗边呼吸下空气保证大脑不缺氧。</li><li>面试前我会买冰水放脸上让自己清醒一点。</li><li>我用的Excel做笔记，track起来挺方便，做完的题就mark成done，看到越来越多的done很开心！</li><li>Excalidraw可以买个会员，这样存在云端比较方便查阅。</li><li>Package negotiate可以在地里看看公司的包裹range，公司的refresh和benefit也要考虑进去 - <a href="https://link.1point3acres.com/?url=https%3A%2F%2Fwww.levels.fyi%2Fcompanies%2Ffacebook%2Fbenefits">https://www.levels.fyi/companies/facebook/benefits</a></li><li>身边有同期在跳槽的朋友最好，可以相互交流和吐槽。</li></ul><br><strong>4. 感言</strong><ul><li>这次跳槽后，以后的工作中会及时总结story + 看技术blog(很多公司的tech blog就挺好)，做终身学习者，而不(止)是城市做题家。</li><li>大部分事都会eventually become easy，5个月前刷题很痛苦，一晚上刷两题，现在一天可以刷40+，无痛刷题；2个月前sd完全不会，现在也能凑合面面。坚持做下去，一定会越来越顺手。</li><li>面试过了不能说明厉害，面试不过也不能说明不行，一半运气，一半努力，运气甚至比努力更重要。</li><li>人再努力也干不过股票，人要踏实，也要会投机取巧。</li><li>爱自己是允许自己失败接受自己失败。</li><li>升不上去不用死磕，不是所有道路都值得坚持，学会适时放弃也是一种能力。</li><li>能走到今天的人都是幸运的， 要把这份幸运传播出去，多做好事攒人品。</li></ul><br>最要感谢家属包揽了大半年的家务！㊗️大家好运！！！<font>. 1point 3acres</font><br>另：答谢地里，可免费帮两位同学mock
